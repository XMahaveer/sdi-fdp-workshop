WHAT SDI IS
Software-Defined Instrumentation replaces racks of fixed-function lab
equipment with one reconfigurable device whose instruments are defined in
software on an FPGA. Our device is the Moku:Go by Liquid Instruments. The
same hardware becomes an oscilloscope, signal generator, or logic
analyzer depending on which instrument is loaded — and everything is
controllable from Python, which is what makes lab automation possible.

OUR HARDWARE: MOKU:GO
- 2 analog inputs (BNC, 30 MHz bandwidth, 125 MSa/s sampling)
- 2 analog outputs (BNC, 20 MHz bandwidth)
- 16 digital I/O pins
- Connectivity: USB-C, Ethernet, Wi-Fi
- Optional programmable power supplies (NOT on all units — never assume)
- Multi-Instrument Mode (MiM): runs exactly TWO instruments
  simultaneously with internal signal routing (no cables between them)

INSTRUMENTS WE USE (Python class names)
Oscilloscope, WaveformGenerator, ArbitraryWaveformGenerator,
LogicAnalyzer, Datalogger, MultiInstrument

KEY PYTHON PATTERNS (simplified, verified on real hardware)
Every script: connect -> configure -> measure -> release.

    from moku.instruments import Oscilloscope
    osc = Oscilloscope('192.168.0.101', force_connect=True)
    try:
        osc.set_timebase(-0.001, 0.001)
        data = osc.get_data()   # dict of Python LISTS: 'time','ch1','ch2'
        vpp = max(data['ch1']) - min(data['ch1'])
    finally:
        osc.relinquish_ownership()   # ALWAYS release the device

Multi-Instrument Mode (two instruments at once):

    from moku.instruments import MultiInstrument, WaveformGenerator, Oscilloscope
    m = MultiInstrument('192.168.0.101', platform_id=2, force_connect=True)
    wg = m.set_instrument(1, WaveformGenerator)
    osc = m.set_instrument(2, Oscilloscope)
    m.set_connections(connections=[
        dict(source='Slot1OutA', destination='Output1'),
        dict(source='Input1', destination='Slot2InA'),
    ])
    ...
    m.relinquish_ownership()   # release via m, never via wg/osc
